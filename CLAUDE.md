## Project Overview

Splitify is a web application that categorizes and organizes Spotify playlists by language, genre, and release era/decade.
It uses Spotify OAuth for login, fetches user playlists (including Liked Songs), and leverages the ChatGPT API for genre classification.
The project name in Gradle is "sorted-music".

## Build & Run Commands

```bash
# Full build (frontend + backend, produces a single JAR)
./gradlew build

# Clean build
./gradlew clean build

# Run the backend (port 8080)
./gradlew bootRun

# Run all tests
./gradlew test

# Run a single test class
./gradlew test --tests "io.jona.smusic.sorted_music.SortedMusicApplicationTests"

# Run a single test method
./gradlew test --tests "io.jona.smusic.sorted_music.SortedMusicApplicationTests.contextLoads"
```

### Frontend commands (run from `frontend/` directory)

```bash
# Install dependencies
npm install

# Start Vite dev server (port 5173, proxies /api to :8080)
npm run dev

# Production build (outputs to src/main/resources/static/)
npm run build

# Lint
npm run lint
```

## Tech Stack

### Backend
- **Java 25** with **Spring Boot 4.0.4** (Gradle Kotlin DSL)
- **Spring Web MVC** (REST API under `/api/**`)
- **Spring Security** (Spotify OAuth integration)
- **Spring Data JPA** + **SQLite** (persistence via `splitify.db`, Hibernate community dialect)
- **Lombok** (annotation-based code generation)
- **JUnit 5** (testing, via `useJUnitPlatform()`)

### Frontend
- **React 19** + **JavaScript** (SPA)
- **Vite 8** (bundler, dev server with HMR, proxy)
- **Tailwind CSS 4** (utility-first styling, configured via CSS `@import "tailwindcss"`)
- **React Router v7** (client-side routing)

### Build Integration
- **Gradle plugin `com.github.node-gradle.node`** orchestrates `npm install` + `npm run build` as part of `./gradlew build`
- Vite outputs the built frontend into `src/main/resources/static/`, which Spring Boot serves as static files
- The final JAR contains both the backend API and the frontend SPA

## Architecture

### Backend
- **Base package:** `io.jona.smusic.sorted_music`
- **Entry point:** `SortedMusicApplication.java`
- **Config package:** `config/` — `SpaController.java` (forwards non-API routes to `index.html`, supports nested routes), `SecurityConfig.java` (OAuth2 login, CSRF disabled for `/api/**`, logout endpoint)
- **Controller:** `controller/PlaylistController.java` — REST endpoints for sync, list, detail, songs
- **Service:** `service/SpotifyService.java` — Spotify API communication, playlist/song persistence
- **Models:** `model/Playlist.java`, `model/Song.java` — JPA entities with SQLite
- **DTOs:** `dto/SpotifyDto.java` (Spotify API response records), `dto/PlaylistDto.java`, `dto/SongDto.java`
- **Repositories:** `repository/PlaylistRepository.java`, `repository/SongRepository.java`
- **Config file:** `src/main/resources/application.properties`
- All REST endpoints go under `/api/**`

### Frontend (`frontend/`)
- **Entry point:** `src/main.jsx` (wraps App in `BrowserRouter`)
- **Router:** `src/App.jsx` (routes: `/login`, `/`, `/playlist/:id`)
- **Pages:** `src/pages/` — `LoginPage.jsx`, `MainPage.jsx`, `PlaylistDetailPage.jsx`
- **Components:** `src/components/` — `SyncOverlay.jsx` (loading modal), `PlaylistCard.jsx` (playlist card)
- **API clients:** `src/services/api.js` — fetch calls for sync, playlists, songs, logout
- **Styles:** `src/index.css` — imports Tailwind CSS
- **Build config:** `vite.config.js` — proxy, output dir, Tailwind plugin

### Key files
- `frontend/vite.config.js` — Vite config (proxy `/api`, `/oauth2`, `/login/oauth2` → `:8080`, build output to `src/main/resources/static/`)
- `build.gradle.kts` — Gradle build with `node-gradle` plugin that runs `npm install` and `npm run build` before `processResources`
- `src/main/resources/static/` — generated by Vite build, gitignored

## Key Domain Concepts (from REQUIREMENTS.adoc)

- **Languages:** Dynamically detected via ChatGPT API (not a fixed list)
- **Genres (10):** Pop, Rock, Hip-Hop/Rap, Electrónica/Dance, R&B/Soul/Funk, Jazz/Blues, Clásica/Orquestal, Reggae/Ska/World, Country/Folk, Latina/Urbano
- **Eras:** by decade (80s, 90s, etc.)
- Filtering supports combinations of language + genre + era
- Song metadata: ID, name, artist, release year, genre, language (genre and language via ChatGPT API — not yet implemented)

## Spotify API Notes

- **Development Mode restrictions:** App can only fetch tracks from playlists the user owns or collaborates on. Followed playlists return 403 on track endpoints. Extended Quota Mode needed for full access.
- **Feb 2026 API changes:** Endpoints renamed from `/tracks` to `/items`. Response field `track` renamed to `item`. Field `tracks` in playlist objects renamed to `items`. DTOs use `@JsonAlias` to handle both old and new field names.

## Important Rules

- **Always stop servers after testing.** When you start `bootRun`, `npm run dev`, or any server process to verify your work, you must stop it (e.g., `taskkill`) before finishing the task. Do not leave servers running.
